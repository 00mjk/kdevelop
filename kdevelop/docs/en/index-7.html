7. The Helpsystem

Quickhelp, Status messages, "What's this ?"

For Version 0.3 of KDevelop, we integrated an extending statusbar message system, displaying statushelp for menubar entries and process status for each selected function. The statusbar always shows you what KDevelop is doing. Furthermore, the statusbar shows the mode of the insert / overwrite key as well as displaying line and comlumn position of the cursor in the editwindow. Dialog fields offer quickhelp by right mouse button click so you don't have to worry about searching functions in the documentation. Also, the "What's this ?" button on the far right of the toolbar offers help window for the main working view. Just push the "What's this ?" button and click on the item you want to have a descriptive help for.

7.1 Using the Documentation Browser

The documentation browser allows quick and easy access to all manuals and documentations, including a search utility that is also present in the editors.

Accessing Help functionality:

  • in the editor, select the text you want to have help on or place the cursor in the word that you want to look for. Then press the right mouse button and select look up: "expression" .This can also be done by selectin "Search Marked Text" from the help menu, the shorcut key F3 or by selecting the Search icon from the toolbar.
  • in the Documentation browser select the text you want to look up and press the right mouse button, select look up:"expression" or select "Search Marked Text" or the search icon as above.
  • if you want to search for a specific keyword, select "Search for Help on..." in the help menu or select the "Search for Help on..." icon from the toolbar, opening a search dialog where you can enter the expression you search for.
Direct access to specific documentation is given by selecting the according icon in the "DOC" -tree-view or by the help menu entries.
